# 题目 4：使用给定数字拼出小于 n 的最大数

## 题目描述
给定一个非负整数 `n` 和一个数字数组 `digits`（元素范围在 `0~9`），你可以从 `digits` 中反复取数字（每个数字可重复使用任意次）来拼接十进制数。  
请返回**严格小于 `n`** 的最大可拼接结果。

## 具体要求
1. 你只能使用 `digits` 中出现过的数字进行拼接。
2. 每个数字可以被重复使用任意次。
3. 拼接出的结果必须满足 `x < n`。
4. 结果不能有前导零；唯一例外是数字 `0` 本身（即 `x = 0` 合法）。
5. 如果无法拼出任何满足条件的数字，返回 `-1`。
6. `digits` 可能有重复项，重复项不应影响结果（按集合语义处理即可）。

## 输入输出说明
- 输入：
  - `n int`：目标上界（非负整数）。
  - `digits []int`：可用数字集合，元素范围 `0~9`。
- 输出：`int`，表示可拼出的最大且小于 `n` 的数字；无解返回 `-1`。

建议函数签名：

```go
func BuildMaxLessThanN(n int, digits []int) int
```

## 示例
### 示例 1（正常场景）
- 输入：`n = 345, digits = [1,3,5]`
- 输出：`335`
- 说明：`345` 不可用（含数字 `4`），在可拼数字中 `335` 是小于 `345` 的最大值。

### 示例 2（正常场景）
- 输入：`n = 572, digits = [2,5,7]`
- 输出：`557`
- 说明：`572` 本身不可取（必须严格小于），最大可行值为 `557`。

### 示例 3（边界：仅 0 可用）
- 输入：`n = 10, digits = [0]`
- 输出：`0`
- 说明：`0` 合法且小于 `10`，不能构造其它合法数字。

### 示例 4（边界：无解）
- 输入：`n = 1, digits = [9]`
- 输出：`-1`
- 说明：无法拼出小于 `1` 的合法数字。
